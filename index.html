Clinical Text De-Identification System

HIPAA-compliant PHI removal system for clinical text using transformer-based NLP models.
Problem: Clinical notes contain sensitive Protected Health Information (PHI) that must be removed before sharing for research or analysis, while preserving clinical meaning.

Solution: Developed an automated anonymization pipeline that identifies and replaces PHI entities with standardized tokens while maintaining text structure for downstream analysis.

Key Features:
• Named Entity Recognition using BERT/RoBERTa transformers for identifying PHI entities
• Custom regex rules for pattern-based PHI detection (dates, IDs, phone numbers)
• Anonymization pipeline replacing entities with tokens: [NAME], [DATE], [ID], [LOCATION]
• High recall on sensitive entity detection ensuring HIPAA compliance
• Preservation of clinical context after de-identification

Tech Stack: Python, HuggingFace Transformers, SpaCy, BERT/RoBERTa, Regex, Pandas

Outcome: Successfully processed clinical documents with high accuracy in PHI detection, ensuring compliance for healthcare data sharing.
